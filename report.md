### Отчёт по итогам тестирования

В период с 31.08.21 по 06.09.21 было проведено интеграционное, функциональное и тестирование пользовательского 
интерфейса вэб-сервиса по покупке тура. 

Была осуществлена подготовка сборки для контейнеризации сервиса в Docker всех 
его компонентов и запуска одной командой, а так же составлена [процедура](https://github.com/m-starilov/diploma/blob/master/readme.md) 
запуска сборки. Написаны и проведены авто-тесты.
По итогам тестирования на непрошедшие проверку тесты были созданы [issues](https://github.com/m-starilov/diploma/issues).
Всего тест-кейсов - 6
1. - [x] Корректное отображение формы при выборе типа покупки
2. - [x] Ввод валидных данных в поля формы
3. - [ ] Ввод невалидных данных в поля формы
4. - [x] Очистка полей формы
5. - [x] Отправка валидных данных
6. - [ ] Отправка невалидных данных4

Процент успешных тест-кейсов 67%, соответственно процент неуспешных 33%.

По итогам тестирования необходимо обратить внимание на корректность вывода сообщения при оплате отклонённой картой, 
так как в БД данные отображаются верно. Так же считаю необходимым добавить проверку данных в поля "Владелец" и 
"Номер карты" дабы исключить отправку некорректных данных в банковские сервисы. Так же прошу обратить внимание на 
таблицу `order_entity` в БД, так как в ней не заполняется поле `credit_id`.
